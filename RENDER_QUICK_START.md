1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click **"New"** ‚Üí **"Blueprint"**
3. Connect your GitHub account if not already connected
4. Select your repository
5. Click **"Apply"**

### Step 3: Wait for Deployment
Render will automatically:
- ‚úÖ Create PostgreSQL database
- ‚úÖ Install Python dependencies
- ‚úÖ Run database migrations
- ‚úÖ Collect static files
- ‚úÖ Start your application

### Step 4: Create Admin User
Once deployed, create a superuser:
1. In Render Dashboard, go to your service
2. Click **"Shell"** tab
3. Run:
   ```bash
   python manage.py createsuperuser
   ```

## Customization

### Change Service Name (Current: glyz-team)
The service is currently configured as `glyz-team`. To change it, edit `render.yaml`:
```yaml
services:
  - name: glyz-team  # Change this to your desired name
    envVars:
      - key: DJANGO_ALLOWED_HOSTS
        value: glyz-team.onrender.com,.onrender.com  # Update this too
      - key: DJANGO_CSRF_TRUSTED_ORIGINS
        value: https://glyz-team.onrender.com  # Update this too

databases:
  - name: glyz-team-db  # Update database name too
```

Then push changes and redeploy.

### Upgrade to Paid Plan
Edit `render.yaml`:
```yaml
services:
  - plan: starter  # or standard, pro

databases:
  - plan: starter  # or standard, pro
```

### Add Custom Domain
1. In Render Dashboard ‚Üí Your Service ‚Üí Settings
2. Click "Add Custom Domain"
3. Enter your domain
4. Update DNS records as shown
5. Update environment variables:
   ```yaml
   - key: DJANGO_ALLOWED_HOSTS
     value: yourdomain.com,glyz-team.onrender.com,.onrender.com
   - key: DJANGO_CSRF_TRUSTED_ORIGINS
     value: https://yourdomain.com,https://glyz-team.onrender.com
   ```

## Troubleshooting

### Build Failed
- Check build logs in Render Dashboard
- Verify all dependencies in `requirements.txt`
- Check Python version in `runtime.txt`

### App Won't Start
- Check application logs for errors
- Verify migrations completed successfully
- Ensure `DJANGO_SECRET_KEY` is set

### Static Files Missing
- Verify `collectstatic` ran in pre-deploy
- Check WhiteNoise is configured in settings
- Review build logs for errors

### Database Connection Failed
- Verify `DATABASE_URL` environment variable
- Ensure database in same region as web service
- Check `DB_SSL_REQUIRE=True`
